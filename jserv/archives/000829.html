使用 XML 技術建構網頁

建構網頁非常簡單，但是要永續經營並且降低維護的複雜度，一直是許多專家不停研究的題目。剛剛找到 piaip 撰寫的一篇紀錄文件 [piaip's XSLT webpage project]，果然獲益良多，真是非常感謝 piaip 無私的分享。

讀到一半，也讓我想到 Slashdot 最近有篇報導 [Printing XML: Why CSS Is Better than XSL]，值得閱讀的部分不僅是新聞所參照到的資料，讀者的評論也很值得一讀，XSLT 實在是太有意思了。

所以我也打算在今年的 TODO List 安插 Web Publishing 的資料涉獵與參考實作。

迴響

我在兩年多前，在等當兵時(那時你好像在花蓮空軍基地當兵)，那時我除了在研究自己包的小型 Linux 之外，也在寫自己的網頁，當時就是覺得網路上充斥著一大堆用 M$ 那不知所云的"所見即所得"的網頁編輯軟體所做的網頁，原始碼只能用"慘不忍睹"來形容，所以我改採 XML + XSL(+CSS) 的形式，成功改寫一個網站的資料(Nue's Lineage http://nue.24cc.com/ 專門介紹線上遊戲"天堂"的網站，其中的 monster 的資料)，該站原來的資料，光 HTML 檔就寫了 3MB，而用 XML+XSL 改寫只要 300K，我覺得 XML+XSL 在資料的呈現真的非常強，只不過原始資料的大小沒辦法跟 IE 的 DataBinding 比。
