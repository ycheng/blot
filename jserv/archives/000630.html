2004 年摩托學園/Debian User 第一次聚會

記錄人：Jim Huang (jserv @ kaffe.org)

聚會公告

從高二開始玩 Linux 以來，還是到了服完兵役才第一次參加 Linux User 的聚會，心中的確是相當的高興。話說敝人是個十足的路痴，早上搭乘捷運坐過頭，心想時間還早，乾脆用走的過去。就這樣，走了近一小時，終於到了傳說中的 [古早傳說景安店]，真是有點距離呀。

打開手機一看，11:28 AM，天呀，真是剛好，於是很自然的就要衝向店裡頭。可是，正要進去的時候才想到，其實今天的聚會雖然很多前輩已經耳聞大名，但是我卻沒有見過除了原真以外的任何人，那我該怎麼辦呢？正當猶豫時，一位笑容可掬的男子開口問說：「來參加聚會的嗎？我是 moto...」，呼，moto 前輩耶，所以我就跟著他上樓了。

到二樓時，發現已經有兩位享用大餐了，一位是日京三子，另一位是他的朋友，他們倆早上十點半就已經前赴聚會場所等候各位朋友的臨駕了。甫坐下，陸續就有一堆朋友加入了，坐我身旁的是 ROXIN666 與 kemlin， 因為我以前也去過 MasterIA (鉅格系統科技)，所以很快就跟 ROXIN666 有了話題。果然 IA 產業變動真是快，除了主管以外，好像我過去認識人都離職了，而向 kemlin 介紹我目前任職的公司時，他竟然念得出公司全銜，原來他的公司跟我們公司有業務來往，真巧。沒多久，一對男女走進來，原來是 AceLan 賢伉儷，AceLan 看起來真是木訥，喔，「大哥的女人」相貌相當標緻，不過我不敢直視，怕誤認為登徒子。後續還多了很多朋友，這裡先行略過...。

回到正題，以下依據殘留的記憶與網友的補充，記錄聚會內容，依據時間順序做介紹：

1. moto

開場白，展示 USB Watch 裡頭如何跑 Linux，相當酷！並且提到 USB Booting 與 BIOS 的問題，因為 moto 任職於 Phoenix BIOS，所以相當清楚這部分。此外，moto 搬來三百本由 LinuxPilot 所贊助的雜誌，所以每個參與聚會的朋友都「滿載而歸」，在此要感謝香港 LinuxPilot (剩下的雜誌就暫時放在店裡)。

在介紹阿信前，moto 語重心長的提到：

「以最近的 "[大怒] Taiwan, Province of China？" 為例，不管是任何議題，我們都希望有多一點 DD 來表達反應我們的心聲，所以誠摯的期待有興趣及時間的朋友來參與...」

按：DD 指的就是 "Debian Developer"，而 moto 所指的事件可參考：
[大怒] Taiwan, Province of China？


2. 阿信

重頭戲，介紹如何成為 DD (Debian Developer)，的確，要成為 DD，實在需要通過重重考驗。阿信說：「要通過五關，但是這五關背後要讀幾百份文件」時，全場幾乎暈倒，DD 真是太偉大了。blue119 提出「DD 讀書會」的構想，聽起來頗有意思，或許腦力激盪的面對面技術切磋可以大幅提昇功力。

有心成為 DD 者應該試著培養社群經驗，並思考種種社群決議的道理，並找到地理位置較為接近的 DD 依據你的功勳為你簽署 PGP Key，或者藉由現有的管道找尋 sponsor，試著包裝套件，貢獻一己之長。


3. 脫線老師

介紹在國小 (主要是脫線老師任教的大豐國小) 推行 Linux 推廣方案，透過 Knoppix 運作 KDE / OpenOffice 來達到桌面系統的需求，似乎相當不錯，脫線老師說小朋友接受度相當高，反而是大人們比較不容易接受。脫線兄在聚會中還抱怨了政府的網站，舉個例子，K12 學習網站可以線上學習，脫線兄本來想去開個 Knoppix 的課程，但是因為 K12 只能使用 Microsoft Internet Explorer，只好作罷。對脫線老師製作的 LiveCD 有興趣者，可以參考Knoppix LTH 版

其實脫線老師學校用的是 DRBL，會作 Knoppix 的原因是：「讓小朋友回家可以練習」，真是為人師表的苦心呀。


4. ROXIN666

介紹 ROXIN666 在 MasterIA 的工作，說明 Linux 在業界的應用，以及他從事的 Kernel porting 工作，相當不錯，有機會一定要多請益。據 ROXIN666 的說法，目前業界 Linux Kernel hacker 的需求頗大，待遇也相當不錯 (看 ROXIN666 就知道了)，所以有心往這方面發展的朋友，可以試著學習研讀硬體規格書以及 RISC CPU 架構。

5. 原真

前輩說話就是不一樣，給予我們對 Embedded Linux 產業相當清楚的分析。其實，要切入 Embedded Linux 並不難，最低的門檻就是 x86 系列的，花不到 NT$1000 就可以買到 32 M 的 Disk-On-Module，就可以在上面放一堆有趣的東西了。另外，更具潛力的解決方案是 RISC CPU，如 ARM 與 MIPS，這也是 Wiscore 正在從事的項目。Linux Kernel 在 RISC 上授權的限制較為鬆散，這也是 Linux Kernel 務實的一面，並沒有特別強調 kernel porting 的廠商一定要 follow GPL，不過基於輿論力量，通常多數的廠商還是會以 patch 的
形式釋出。

6. moto

回頭介紹他任職的 Phoenix BIOS 公司，以及目前從事的項目，真令人大開眼見。Phoenix BIOS 自從併購 Award BIOS 後，便積極展開許多進階的應用，
moto 舉出在他們的研發團隊正設計出一種獨特的架構，可以在 BIOS 直接切割
硬碟空間，並保留該區域當作備份、救援，或是 3-rd party 廠商內建工具，已經有許多 Notebook 廠商向 Phoenix 接洽授權事宜。moto 舉出該公司的個案，的確是很有意思的嵌入式系統應用方式。

7. 原真

補充對 Embedded Linux 的分析，以業界對於 Embedded Linux 人才的需求來說，只要「有一技之長，不必擔心沒頭路」。所以呀，各位 Debian 愛好者，
用力的玩吧 :-)

8. moto

總結，並且提到 DWN (Debian Weekly News) 的中文翻譯，希望社群能有更多熱情的朋友協助參與。moto 同時也希望能有更多學生身份的社群朋友投入
Debian@Taiwan 的活動，像是翻譯文件、包裝套件，甚至往 DD 之路邁進。

還有，moto 提到今年 Aug 10 是 Debian 生日，雖然可能沒有像去年十週年的盛大活動，但是有可能再度辦場聚會 (所以應該是第二次聚會的時間) ，請各位同好耐心期待。

當然，天下沒有不散的宴席，moto 指出，當天聚會完立刻趕到員林「娘家」 (按：老婆的母親家裡) 去過母親節，所以本次聚會大約進行到下午四點二十分。

嗯，對於我這個「新鮮人」來說，最有趣的部分是自我介紹，看到許多 id 的本尊，還真是驚豔呢。對了，今天本來準備了一些關於 Free JavaVM 與中文相關的議題，但是自我介紹的時候，moto 太熱情的介紹我，所以避免話題重複，就跳過了。等有空我再來完成《淺談 JavaVM 發展》一文，並且敘述當前中文相關的議題。KaffeVM 是 Debian 提供的 Java Runtime 運作最好的，實際上已經可以做很多應用了，但是中文問題不僅有技術層面 (我已經處理一部份)，還有 policy 問題，類似之前的 DI (Debian Installer) 問題。在 Sun Microsystem 的 JDK 中，台灣是視為一個國家，但是對岸許多有心人士以為數眾多的 Bug report 反映，所以，很可能 Sun 會更改其實作，那麼，Free JavaVM 實作就有可能會跟進，身為 Kaffe Developer，我要以何種態度、何種方式處理呢？就算 Sun 因為對岸朋友的抗議改變實作，但是 Kaffe 是 Free 的，是自由的，不應該左右搖擺，對吧？

未討論的項目是：要透過什麼機制，來分享各位 Debian User 所打包的特殊版本套件，比方說加入 FireFly 的 patch、修正中文問題，或是加入其他修改呢？

moto 提出本次聚會美中不足之處以及有趣的插曲：

「這次的地點真的不是很好，需要好好的檢討，除了煙霧瀰漫，還有人聲鼎沸真的不是一個好的聚會場所... 對了，聚會過程中有一段的小插曲，就是當天恰巧是有另外一個好像是什麼"大貓"的網路聚會活動也在同一地點舉辦，有一位"大貓"聚會的大哥(年約 40 幾) 走錯地方，我還跟他握手問候了許多，有點離譜的是他還坐下來聆聽我們的報告，我看大哥一臉疑惑，仔細瞭解之後才發現是跑錯場子了... 後來陸續還有兩三位類似的案例，我也都很有經驗地在第一時間把"大貓"聚會的正確包廂指了出來.. 真是有趣的插曲...」

以上，感謝 moto 以及各位共赴盛會的朋友，這次的聚會真是相當棒，咱們八
月份再見面了！

- Jim Huang
