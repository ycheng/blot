qemu 使用感

最近看到了一個軟體或應該稱為模擬器：qemu

安裝是滿簡單的，只要 aptitude install qemu，然後建立一個 disk image就可以了：
qemu-disk create img 4G
qemu -cdrom ~/tmp/woody.iso -hda debian.img -boot d

這樣就能光碟開機並安裝作業系統了，安裝 w2k 和 wxp 也是可以，但我覺得很慢...Orz...安裝 Debian 的部份呢，還算可以，速度可以接受。

我發現似乎在 disk 讀寫時會慢很多，或許是因為沒有 kqemu 這個外掛的 kernel module，所以速度上慢許多，而 maintainer 也不打算包裝這個 kqemu(non-free issue)。

還有一個 qemu-fast 指令，但我就是無法啟動，真奇怪...

Anyway...可以嘗鮮看看...:-D

udev and multidevice(aka Linux SoftRaid)

目前 kernel-2.6.8(below) 搭配 udev 的話，無法自動產生 md node，必須手動建立 device node(其實是有 dirty hack 的...:-p)，請修改 /etc/udev/links.conf

# MD nodes
M md2 b 9 2
M md1 b 9 1
M md0 b 9 0

重跑 /etc/init.d/udev 或 reboot 就會看到 /dev/mdX 的node。附註一下，這是 raid 編譯成 modules 的情況，如果編譯到 kernel 之中，會自動產生。但是！Debian 預設包裝的是以 modules 為主。

附註二：如果您使用 kernel-2.4 + devfsd，forget about the article. :-D
