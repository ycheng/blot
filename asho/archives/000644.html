Raid 卡在 Linux上的支援

這是在 BBS 上小弟 post 的文章，讓大家參考看看目前 raid 卡支援的程度。

關於 acard aec-6880 raid 卡官方說在linux下只能使用soft raid,且無法認出該卡 bios 所建立
得 raid partition。所以說, acard 6280 = aec 6880 in Linux
但小弟對該家的產品不是很喜歡,因為都不能提供 redhat/suse 以外的driver
請他們提供 debian 得 driver 被拒絕...:-(


目前只有 3ware 是pure hardware raid

high-point/promise/si 也是軟體raid,只是 linux
能讀寫raid 卡上得bios所建立的 raid 磁區，但讀寫
依然是 linux soft raid。(Kernel 說明因為這樣能提高穩定度以及效能)

您可以參考 kernel source 中關於 ATA soft raid 以及 3warexxxx.h
得 source code 或是 doc


阿...忘了說明,si3112/4a 以及 siw680 都只有支援 raid 0喔
不支援 raid 1. 如果預算無問題的話,可以考慮 3ware 的產品
如果有預算,可以買 high point.有提供kernel 2.4/2.6 x86 64

而Promise 只有kernel2.4 的而已，有點可惜...

BTW.如果您的系統只有 Linux，可以考慮用 soft raid，效能上也不輸給
卡上的bios喔...:-)

------補充

最近另外買了一張 ite8212f的 raid 卡，這個晶片常常出現在 gigabyte or ASUS 的主機板上，廠商有提供 source code可以下載且可以讀寫卡上bios所建立的raid 磁區，該卡價格在 650左右，算是經濟又實惠...:-)
