反制SPAM的第一步

四月十五日起，各大 ISP 將共同採行拒絕接收動態IP架設郵件伺服器直接寄送之信件。這是反制 SPAM 的一個里程碑，但真正的戰役，才正要開始……

聽到這消息我很高興，因為多年來的垃圾郵件問題，國內網路供應商終於在這一天取得共識，並聯手反制。雖然這樣的防堵策略並不能完全解決，但卻是各 ISP 間合作反制 SPAM 的良好開始。

我相信，這樣的反制措施，不僅是針對垃圾郵件，對病毒防護也有相當大的幫助。因為越來越多的病毒，利用自己的 SMTP 服務，從感染者的電腦發送病毒信件，不知不覺中，自己的親朋好友也因此中毒。而目前台灣多數的家庭上網方式都是 ADSL 動態動態 IP ，我認為這樣的防堵措施，更能夠在使用動態 IP 的一般用戶，不幸感染病毒時，控制損害範圍。

所以，雖然這不是完美的解決方案，但是我肯定這樣的作法。當然，我更希望這是 ISP 之間，正視垃圾郵件問題，並通力合作的起步。

水的突沸現象

「突沸」是因為液體溫度已經達到沸點，但卻沒有沸騰所造成。這個時候，如果突然改變液體環境，例如攪拌水、搖晃杯子，或像上文中，把杯子從微波爐拿出來的話。液體將會立刻沸騰，並瞬間變態氣化，造成液體往上噴發。這就是為什麼他說：「水都飛濺到他的臉上」的原因，那就像把臉直接對著燒開的水壺口一樣。慘……/_\

而然，並不是「水絕對不能在微波爐中單獨加熱」。我們都知道，水沸騰時，都會產生許多氣泡形成蒸汽並帶走能量，這些氣泡之所以會產生，則是因為液體中的「氣化核」所形成。造成「突沸」的原因，就是因為液體中沒有適當的氣化核，比如太平滑乾淨的杯子，無雜質的純水。所以上文中後來提到可以放置「木製的攪伴棒或茶包」，其實主要並不是要用來擴散熱量，而是要讓液體有足夠的氣化核，形成沸騰。這包括表面粗糙的容器，或者灑點糖或鹽，都是可以的。總之，當液體中有適當的氣化核，就可以在沸點時形成氣泡造成沸騰，也就不會發生突沸現象。所以液體還是可以在微波爐中加入。但是，微波爐加熱液體最好不要過長，因為它功率高加熱速度快，比較容易造成突沸現象。

GnuPG's ElGamal signing keys compromised

GnuPG.org 在 2003-11-27 緊急發佈一則 GnuPG 重大安全漏洞的警訊，請求使用者立即廢止有使用 ElGamal signing keys 的金鑰。

法國的一位研究員 Phong Nguyen 已經首先證實，GnuPG 在建立並使用 ElGamal Keys 簽署的過程中，存在一個嚴重的安全漏洞！這個嚴重的漏洞會導致 private key，在數秒的時間內被破解。當你的金鑰有使用 ElGamal sign+encrypt keys (type 20) 時，也就是你的 ElGamal 可同時用來加密和簽署，那這個問題就會存在。

由於 GnuPG >= 1.0.2 的版本才能製作 type 20 的金鑰。換句話說，由 GnuPG < 1.0.2 的版本所製作的金鑰並不會有這問題。

如何檢查是否為 type 20 金鑰：
1. 使用 gpg --list-keys 來檢視自己的金鑰資料。
$ gpg --list-keys xxxxxxxx

2. 如果你金鑰類型部分是大寫的 "G" ，例如下列 "2048G" ，那麼就表示這把金鑰是一支 type 20 的金鑰。你應該立即停用並發佈廢止通知。
pub 2048G/xxxxxxxx 2001-xx-xx Mallory

PS. 如果是小寫的 "g"，那這是一支標準的 ElGamal encrypt-only keys (type 16)。只能用來加密，並不會構成加密環節的威脅。

！！只有大寫的 "G" 必須立即廢止！！

Docbook 文件製作環境

要在 Linux 下建立一個 Docbook 的出版平台，似乎比 Windows 容易許多...

Docbook 是文件出版用的描述語言標準，Docbook 最初以 SGML 為主要描述語言，到 Docbook 4.1 後加入 XML 的支援，並開始以 XML 為建議使用的描述語言。使用 Docbook 的好處在於，一份使用 Docbook 製作的文件檔，可以輸出成多分不同的出版格式，包括 plain text，HTML，PDF...等等。而且根據不同場合需要，只要變換 Stylesheets ，就可達成不同的顯示樣式。換句話說，Docbook 提供的是文件的內容，但並不描述樣式，文件該長什麼樣子，是由 Stylesheets 來描述。
要建立一個 Docbook 文件的出版環境，需要三樣東西，Docbok DTD，Docbook Stylesheets 和 XSLT Engine。有了這三樣東西，就可以製作交換行強的文件檔案。
